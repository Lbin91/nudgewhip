# NudgeWhip v0.2.0

First stabilization release.

This release keeps the macOS core loop intact while tightening the roughest edges from the initial beta.

## Highlights

- New statistics dashboard focused on recovery, not punishment
- Focus-session app usage tracking for privacy-safe "Top Apps" insights
- More reliable onboarding recovery for Accessibility-denied users
- Cleaner settings flow with update check and release-distribution groundwork
- Stabilized menu bar presentation and less misleading unfinished pet surface area

## What's New

### Recovery-first statistics dashboard

`0.2.0` adds the first real detailed stats surface for NudgeWhip.

You can now inspect:

- total focus time
- alert count
- recovery rate
- longest focus block
- average recovery time
- focus trend across today / this week / last 7 days
- top apps used during focus sessions

The dashboard is intentionally framed as a recovery coach, not a surveillance report.

### App usage measurement inside focus sessions

NudgeWhip now records privacy-safe app usage segments during active focus sessions.

That enables:

- Top Apps during focus
- primary app summaries
- transition counts for context switching

What still does **not** happen:

- no window title inspection
- no URL inspection
- no typed text capture
- no terminal contents/history capture

### Better onboarding and limited-mode recovery

The first-run flow is more stable and clearer than in `0.1.0`.

Changes include:

- clearer limited-mode completion copy
- stronger permission recovery path from onboarding and settings
- better handling when the user grants Accessibility after first denial
- improved continuation flow between welcome, permission, setup, and completion steps

### Settings and distribution polish

`0.2.0` also cleans up the surrounding product surface:

- lighter settings structure
- release/update lane prepared around Sparkle + GitHub release distribution
- countdown overlay position controls
- better release metadata and notarized DMG flow

### Reduced unfinished surface area

This release intentionally trims back product areas that were visible in code or docs but not truly product-ready.

Most notably:

- unfinished pet progression/presentation paths were removed from the active product surface
- the existing Little Devil presence in onboarding and the menu bar remains

## Improvements

- one-shot idle deadline scheduling remains the core detection model
- break suggestion actions are safer and focus on reducing alert fatigue without forcing break mode
- repeated warning copy is less repetitive
- visual alert behavior better respects accessibility settings
- menu bar interactions remain aligned with the non-regression guard around pause controls

## Fixes

- fixed bundle-version progression for Sparkle release metadata so `0.2.0` is no longer ambiguous with `0.1.0`
- fixed release appcast metadata to match the final notarized and stapled DMG
- fixed UI test isolation so stored local settings no longer bleed into onboarding test runs

## Distribution Notes

`0.2.0` is distributed as a notarized GitHub release DMG for direct download on macOS.

This is a notarized Developer ID release, not a Mac App Store review release.

## Out Of Scope

- iOS companion app
- CloudKit sync
- Pro packaging
- expanded progression systems
