# NudgeWhip v0.4.0

iOS companion app and CloudKit real-time sync.

This release introduces the iPhone companion app that receives live focus state and escalation alerts from your Mac via CloudKit, along with multi-device isolation, full Korean localization, and content planning for the upcoming pet system.

## Highlights

- iOS companion app with live CloudKit sync for Mac focus state
- Multi-Mac isolation — correctly handles multiple Macs sharing one iCloud account
- Full Korean (한국어) localization for both macOS and iOS
- Remote escalation event pipeline from macOS to iOS
- Pet content strategy and character design documentation for the next release

## What's New

### iOS Companion App

NudgeWhip now has an iPhone companion that shows real-time Mac focus state via CloudKit Private Database sync.

The iOS app displays:

- current Mac state (monitoring, idle, alerting, recovery, break)
- escalation alert history with timeline
- daily and weekly focus statistics with trend charts
- connection status and last sync timestamp
- settings for sync interval and notification preferences

What still does **not** happen:

- no push notifications yet (CKQuerySubscription + APNs planned for next release)
- no bi-directional control (iOS cannot change Mac settings)
- no standalone iOS focus tracking

### CloudKit Real-Time Pipeline

macOS now writes two record types to CloudKit Private Database on every relevant state change:

- **MacState**: current runtime state, idle countdown, escalation step, last activity timestamp
- **RemoteEscalationEvent**: timestamped escalation events with step and content state

Both records are scoped by `macDeviceID` to support multiple Macs under one iCloud account.

### Multi-Mac Isolation

All iOS queries are scoped by `macDeviceID`. When multiple Macs write to the same CloudKit container:

- each Mac's state is cached separately (`CachedMacState` keyed by `macDeviceID`)
- daily stats projections use composite keys (`macDeviceID + date`)
- the iOS app auto-detects the most recently active Mac
- escalation alerts are attributed to the originating Mac

### Korean Localization

Both platforms now have complete Korean translations:

- macOS: 9 new xcstrings keys translated, 4 hardcoded `Text()` calls replaced with `String(localized:)`, 2 accessibility labels localized
- iOS: `RelativeDateTimeFormatter` for relative timestamps, `DateComponentsFormatter` for duration display, locale-aware time formatting

### Pet Content Strategy

Documentation for the upcoming pet system:

- character world-building, personality, and growth narrative
- mood system with 6 emotional states and state transition table
- growth stage definitions with story beats
- sprite asset plan and production guide
- content tone guardrails (no guilt-tripping, recovery-oriented messaging)
- phased implementation roadmap (v1: data model + text, v1.1: assets + interaction)

See `docs/content/` for full details.

## Improvements

- timestamp-based sequence IDs (`Int64(date.timeIntervalSince1970 * 1000)`) survive app restarts instead of in-memory counters
- `saveRemoteEscalationEvent` now passes the actual event timestamp instead of creating a new `Date()`
- `MacRuntimeState` shared constants enum replaces raw string comparisons on iOS
- `PetProgressionService` and `PetState` models in place for future pet features

## Fixes

- fixed `limitedNoAx` typo — now `limitedNoAX` consistently across macOS and iOS
- fixed `AlertsViewModel` dead code and missing `timeStyle` locale configuration
- fixed `StatsViewModel` duplicated query code
- fixed `.gitignore` to track `docs/content/` directory

## Distribution Notes

`0.4.0` is distributed as a notarized GitHub release DMG for macOS.

The iOS companion is not yet on the App Store. It runs from Xcode directly on registered devices.

This is a notarized Developer ID release, not a Mac App Store review release.

## Out Of Scope

- Pet mood system implementation
- Pet sprite assets and animations
- Pet interactions (pat, feed, play)
- StoreKit / Pro paywall
- Recovery Review dashboard
- Schedule presets
