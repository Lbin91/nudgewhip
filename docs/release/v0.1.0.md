# NudgeWhip v0.1.0

First public beta.

This release establishes the core NudgeWhip loop on macOS: detect idle moments, respond locally, and help the user return to work without punitive UX.

## Highlights

- Menu bar app with `LSUIElement` behavior
- Accessibility permission onboarding with limited-mode fallback
- Global input-based idle detection
- One-shot idle deadline scheduling
- Local visual nudge flow
- System notification escalation
- Runtime status and countdown in the menu bar dropdown
- Schedule controls
- Local daily summary stats
- Settings window and launch-at-login toggle
- Korean and English support

## Positioning

NudgeWhip is not a blocker.

It is an `attention recall tool` for people doing serious work on a Mac. The product is designed around a simple premise: distraction is normal, recovery is the moment that matters.

## What Is Not In Scope Yet

- iOS companion app
- CloudKit sync
- Pro packaging
- richer exception handling and advanced whitelist workflows
- expanded progression systems

## Privacy

- Accessibility permission is used only for global input activity detection
- No typed text is read
- No screen content is collected
- No browsing history is collected
- Summary data is stored locally

See: [accessibility-and-data-disclosure.md](../privacy/accessibility-and-data-disclosure.md)

## Technical Baseline

- Platform target: `macOS 15+`
- Primary stack: `SwiftUI`, `AppKit`, `SwiftData`

## Verification Notes

Verified during release prep:

- `xcodebuild build -scheme nudge -destination 'platform=macOS'`
- `xcodebuild test -scheme nudge -destination 'platform=macOS' -only-testing:nudgeUITests`

Known note:

- UI testing for menu bar agent apps is more timing-sensitive than standard foreground apps, so test harness stability work is still ongoing.
