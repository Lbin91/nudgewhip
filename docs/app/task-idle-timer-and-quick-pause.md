# Feature Spec: Idle Stopwatch & Overlay Quick Pause

- **Version**: v1.0
- **Status**: Draft
- **Owner**: Product Management
- **Target Version**: v0.3.x

## 1. 개요 (Executive Summary)

NudgeWhip의 핵심 가치는 사용자가 흐트러진 순간을 빠르게 인지하고 업무로 복귀하도록 돕는 것입니다. 현재의 오버레이 알림은 "업무 이탈" 사실만을 알리고 있으나, 사용자는 **"정확히 얼마나 자리를 비웠는지"**에 대한 실시간 정보와, 상황에 따라 **"알림을 유연하게 제어할 수 있는 방법"**을 요구하고 있습니다.

본 기획은 알림 화면에 실시간 스톱워치를 도입하여 이탈 시간의 체감도를 높이고, 오버레이 내에서 즉시 일시정지(Pause)가 가능한 퀵 메뉴를 추가하여 제품의 사용성을 개선하는 것을 목표로 합니다.

---

## 2. 핵심 기능 1: 실시간 자리비움 타이머 (Idle Stopwatch)

### 2.1 기획 의도
- **심리적 넛지**: "5분 지남"이라는 정적인 문구보다, 1초씩 올라가는 타이머가 사용자의 복귀 본능을 더 강력하게 자극합니다.
- **데이터 신뢰성**: 사용자가 자리에 돌아왔을 때, 자신이 얼마나 자리를 비웠는지 즉각적인 피드백을 제공하여 통계 데이터와의 연결감을 강화합니다.

### 2.2 상세 사양
1.  **시작 시점 (Escalation Policy)**: 
    - **1단계 알림**: 기존처럼 정적인 문구(예: "Are you there?")만 표시하여 가벼운 주의 환기.
    - **2단계 알림 (표시 시작)**: 자리비움이 지속되어 2단계로 격상되는 즉시 실시간 타이머 노출.
2.  **표시 내용**: `이탈 시작 시점의 누적 시간` + `실시간 흐르는 시간`.
    - 예: 5분 간격 설정 시, 2단계 진입 시점인 **"10:00"**에서 시작하여 실시간 카운트업.
    - 1단계(0~10분) 동안 기록된 이탈 시간을 소급하여 보여줌으로써 사용자에게 "이미 10분이 지났다"는 사실을 강력하게 인지시킴.
3.  **UI 디자인**:
    - 위치: 오버레이 중앙 문구(예: "Are you there?") 하단 또는 상단에 배치.
    - 스타일: 가독성이 좋은 모노스페이스(Monospace) 폰트 권장 (숫자 너비 변동으로 인한 UI 흔들림 방지).
    - 컬러: 강조색(Accent Color)을 사용하되, 시간이 길어질수록(예: 15분 이상) 색상을 점진적으로 변화(예: White -> Yellow -> Light Orange)시키는 시각적 압박 고려.

## 3. 핵심 기능 2: 오버레이 퀵 일시정지 (Quick Pause)

### 3.1 기획 의도
- **이탈 방지**: 알림이 업무 흐름을 방해한다고 느낄 때 사용자가 앱을 완전히 종료해버리는 '최악의 시나리오'를 방지합니다.
- **상황 대응**: 갑작스러운 회의, 전화, 휴식 등 정당한 이탈 상황에서 메뉴바까지 가지 않고도 오버레이 내에서 즉시 "합법적 휴식"을 선언하게 합니다.

### 3.2 상세 사양
1.  **진입점 (Entry Point)**:
    - 오버레이 하단 또는 모서리에 `Square Shape`의 일시정지(Pause) 버튼 배치.
    - 아이콘: 일반적인 일시정지(`||`) 또는 정지(`■`) 기호 사용.
2.  **상호작용 (UX Flow)**:
    - 버튼 클릭 시, 버튼 위쪽으로 **수직 리스트(Pop-up Menu)** 노출.
    - 옵션 구성: `10m`, `30m`, `1h`, `Until Tomorrow`, `Custom...`.
    - 선택 즉시: 해당 시간만큼 일시정지 모드 진입 및 오버레이 즉시 종료.
3.  **UI/UX 고려사항**:
    - **Hover State**: 버튼에 마우스를 올리면 "Quick Pause" 툴팁 또는 강조 효과 제공.
    - **Safety**: 실수로 클릭하는 것을 방지하기 위해 버튼 크기는 너무 크지 않게 조절하되, 리스트의 클릭 영역은 충분히 확보.
    - **Animation**: 리스트가 아래에서 위로 부드럽게 슬라이드하며 나타나는 연출.

## 4. 사용자 시나리오 (User Scenario)

1.  **이탈 인지**: 사용자가 유튜브를 보느라 5분이 흐름 -> 오버레이 발생.
2.  **시간 체감**: 오버레이 중앙에 `05:12... 05:13...` 올라가는 타이머를 보고 "아, 벌써 5분 넘게 딴짓했네"라고 인지함.
3.  **상황 판단**: 마침 팀원에게 전화가 옴. 지금은 알림을 끌 수 없는 상황.
4.  **퀵 액션**: 오버레이 우측 하단의 [■] 버튼 클릭 -> `30m` 선택.
5.  **상태 전이**: 앱은 30분간 일시정지 모드로 들어가고, 오버레이는 사라짐. 사용자는 미안함 없이 통화에 집중.

---

## 5. 기대 효과 (Expected Impact)

- **Retention**: 알림의 피로도를 유연하게 관리하게 함으로써 앱 삭제/종료율 감소.
- **Engagement**: 사용자가 자신의 시간 낭비를 초 단위로 직면하게 하여 업무 복귀율(Recovery Rate) 향상.
- **UX Polishing**: 메뉴바를 통하지 않는 직관적인 제어로 고급스러운 유틸리티 앱 경험 제공.

---

## 6. 개발 가이드 (Implementation Note)

- **Timer logic**: `MainActor`에서 동작하는 `Timer.publish`를 활용하여 UI 업데이트.
- **Pause logic**: 기존 `RuntimeStateController`의 `pause(duration:)` 메서드 재활용.
- **Component**: SwiftUI의 `Menu` 또는 `Popover`를 활용하여 퀵 메뉴 구현 가능성 검토.
