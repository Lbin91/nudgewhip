## 1. 개요

### 1.1 Phase 1 목표

v1.0 정식 버전 출시를 위해 macOS 코어 루프의 완성도를 높인다. v0.3에서 확인된 세 가지 가치 축—Trust, Reflection, Retention—을 실제 기능으로 구현한다.

### 1.2 범위

| ID | 기능 | 우선순위 | 복잡도 |
|---|------|---------|--------|
| F1 | Recovery Review v1 | P0 | Medium |
| F2 | Schedule & Pause Presets | P0 | Medium |
| F3 | Pet Progression v1 | P1 | Medium |

### 1.3 범위 밖

- iOS companion 본격 구현
- StoreKit / Pro paywall
- Calendar 연동 자동 pause
- Browser/domain 수준 화이트리스트
- Weekly Insight Cards, Nudge Tone Packs (Phase 1 이후 capacity 여분 시 검토)

### 1.4 마일스톤

| Milestone | 내용 | 선행 조건 |
|-----------|------|-----------|
| M1 | F1 Recovery Review v1 | 없음 |
| M2 | F2 Schedule & Pause Presets | 없음 (M1과 병렬 가능) |
| M3 | F3 Pet Progression v1 | F1, F2 완료 후 (XP 소스가 recovery/session에 의존) |
| M4 | 통합 검증 + 로컬라이제이션 마무리 | M1~M3 완료 |

## 2. F1: Recovery Review v1

### 2.1 기능 요약

**한 줄**: 오늘/이번 주에 언제 흐름이 끊겼고 얼마나 빨리 복귀했는지 이해할 수 있는 전용 회고 뷰.

**사용자 가치**:
- 알림이 왜 왔는지, 어느 시간대에 집중이 자주 깨지는지 스스로 파악 가능
- 복귀 속도 변화 추이로 넛지가 실제로 도움이 되는지 확인 가능
- raw log가 아닌 요약된 회고 형태로, 심문이 아닌 짧은 돌아보기 경험

### 2.2 UX 흐름

```
[Settings 통계 탭] → "복귀 리뷰" 버튼/탭
  → RecoveryReviewView 진입
    ├─ 상단: 기간 선택 피커 (오늘 / 이번 주 / 최근 7일)
    ├─ 요약 카드 (3장)
    │   ├─ "가장 빠른 복귀" — 최소 recoveryDuration
    │   ├─ "가장 긴 이탈" — 최대 recoveryDuration (미복구 포함)
    │   └─ "가장 산만한 시간대" — alertCount 최대 hour
    ├─ 시간대별 히트맵
    │   └─ 24시간 기준 바/셀, 색상 농도로 이탈 빈도 표현
    └─ 복귀 타임라인
        └─ 날짜 역순 이벤트 리스트
            ├─ alerting 시작 시각
            ├─ 에스컬레이션 단계 (1/2/3)
            ├─ 복귀 시각 + 소요 시간 (또는 "미복구")
            └─ 세션 정보 (해당 FocusSession 요약)
```

**빈 상태**: "아직 복귀 기록이 없습니다. 몇 번의 포커스 세션을 완료하면 여기에 복귀 리뷰가 나타납니다."

**접근성**: Reduce Motion 시 히트맵 애니메이션 대신 정적 색상 렌더링. 색상 외에 숫자 라벨 병기.

### 2.3 데이터 모델 변경

**새로운 @Model 없음.** 기존 데이터로 충분:

| 데이터 소스 | 사용 필드 | 용도 |
|------------|----------|------|
| `AlertingSegment` | startedAt, recoveredAt, maxEscalationStep | 개별 복귀 이벤트 |
| `FocusSession` | startedAt, endedAt, alertCount, recoveryCount | 세션 컨텍스트 |
| `DashboardDayProjectionPayload.hourlyAlertCounts` | [Int] (24 elements) | 시간대별 히트맵 |

**새로운 값 타입** (SwiftData @Model 아님):

```swift
struct RecoveryEvent: Identifiable, Sendable {
    let id: UUID
    let alertStartedAt: Date
    let recoveredAt: Date?

    let recoveryDuration: TimeInterval?          // nil = 미복구

struct RecoverySummary: Sendable {

    let longestDistraction: TimeInterval?
    let mostDistractedHour: Int?                  // 0-23
    let totalEvents: Int
    let recoveredCount: Int
    let unrecoveredCount: Int
}
```

## 3. F2: Schedule & Pause Presets

### 3.1 기능 요약

**한 줄**: 출근/야간/집중시간 등 자주 쓰는 운영 모드를 빠르게 전환하고, 일시정지도 프리셋 기반으로 선택.

**사용자 가치**:
- 반복적인 스케줄 설정을 1클릭으로 해소
- 메뉴바에서 즉시 프리셋 전환 → 설정 창 열 필요 없음
- 일시정지 시간도 빠른 선택 → 매번 수동 입력 불필요

### 3.2 UX 흐름

#### 스케줄 프리셋

```
[MenuBarDropdownView] → QuickControlsView
  → 현재 활성 프리셋 이름 표시 (또는 "사용자 정의")
  → 프리셋 선택 드롭다운/피커
    ├─ "평일 집중" (09:00–18:00, 평일만)
    ├─ "야간 모드" (22:00–06:00, 매일)
    ├─ "하루 종일" (00:00–23:59, 매일)
    ├─ ──────────
    ├─ "+ 새 프리셋" → SchedulePresetEditorView
    └─ "프리셋 관리" → SchedulePresetListView

[SettingsRootView] → Schedule 섹션 확장
  → SchedulePresetListView
    ├─ 내장 프리셋 (수정 불가, 시간만 표시)
    ├─ 커스텀 프리셋 리스트 (편집/삭제 가능)
    └─ "+ 새 프리셋 추가" 버튼
  → SchedulePresetEditorView (시트)
    ├─ 프리셋 이름 입력
    ├─ 시작/종료 시간 피커
    ├─ "평일만" 토글
    └─ 저장/취소
```

#### Pause 프리셋

```
[MenuBarDropdownView] → Pause 버튼 롱프레스 또는 확장

    ├─ "15분"
```

## 4. F3: Pet Progression v1

### 4.1 기능 요약

**한 줄**: 복귀와 세션 완료를 기반으로 성장하는 가상 펫으로, 켜 둘 이유와 감정적 애착 형성.

**사용자 가치**:
- 모니터링 유지에 대한 시각적 보상
- 성장 단계 전환의 달성감
- 펫 상태 확인하러 앱을 여는 자연스러운 접근

### 4.2 UX 흐름

#### 메뉴바 요약

```
[MenuBarDropdownView] → 하단 펫 요약 카드
  ├─ 펫 아이콘 (현재 단계 SF Symbol)
  ├─ 펫 이름
  ├─ 현재 단계 라벨 (예: "아기 알")
  └─ XP 진행 바 (다음 단계까지 %)
  → 탭 → PetDetailView 열기
```

#### 펫 상세 뷰

```
[PetDetailView]
  ├─ 펫 아이콘 (큰 버전)
  ├─ 펫 이름 (편집 가능)
  ├─ 현재 단계 + XP 프로그레스 바
  │   ├─ "아기 알" ━━━━━━░░░ 72%
  │   └─ 다음 단계: "새끼" (50 XP 필요)
  ├─ 성장 단계 타임라인
  │   ├─ 🥚 아기 알 (0–49 XP) ✓
  │   ├─ 🐣 새끼 (50–149 XP) ← 현재
  │   ├─ 🐥 어린이 (150–349 XP)
  │   ├─ 🐤 청년 (350–699 XP)
  │   └─ 🦅 장년 (700+ XP)
  ├─ 통계 카드
  │   ├─ 총 복귀 기여 횟수
  │   ├─ 연속 활성 일수
  │   ├─ 함께한 일수
  │   └─ 총 획득 XP
  └─ 초기화 버튼 (경고 다이얼로그 후)
```

**초기 설정**: 첫 실행 시 펫 자동 생성 (이름: "Whip", 단계: egg). 펫 이름은 설정에서 변경 가능.

### 4.3 데이터 모델 변경

#### 신규 @Model: `PetState`

```swift
enum PetGrowthStage: String, Codable, CaseIterable, Sendable {
    case egg        // 0–49 XP
    case hatchling  // 50–149 XP
    case juvenile   // 150–349 XP
    case adult      // 350–699 XP
    case elder      // 700+ XP

    var displayName: String {
        // localized key로 변환
    }

    var xpThreshold: Int {
        switch self {
        case .egg: return 0
        case .hatchling: return 50
        case .juvenile: return 150
        case .adult: return 350
        case .elder: return 700
        }
    }

    var nextStage: PetGrowthStage? {
        switch self {
        case .egg: return .hatchling
        case .hatchling: return .juvenile
        case .juvenile: return .adult
        case .adult: return .elder
```

#### XP 보상 규칙

| 이벤트 | XP | 조건 | 비고 |
|--------|-----|------|------|
| 복귀 성공 | +5 | AlertingSegment.recoveredAt != nil | SessionTracker에서 트리거 |
| 세션 완료 (30분+) | +10 | FocusSession.contributesToFocusTotals && duration >= 1800 | SessionTracker에서 트리거 |
| 일일 활성 보너스 | +2 + streak bonus | 하루 최초 활동 시 | streak bonus = min(consecutiveDays, 5) |
| **감소/벌점** | **없음** | **절대 없음** | punish 금지 원칙 |

**밸런싱 원칙**: XP 보상 상수(+5/+10/+2, streak bonus)와 단계 임계값(50/150/350/700)은 `PetProgressionConstants.swift`에 별도 분리한다. 향후 밸런싱 시 코드 수정 없이 상수 파일만 조정으로 가능하게 한다.

**스트릭 계산**:
- `lastActiveDate`가 오늘이면 streak 유지
- `lastActiveDate`가 어제면 streak + 1
- `lastActiveDate`가 어제 이전이면 streak 리셋 = 1
- 하루에 여러 번 활동해도 streak은 1 증가

## 7. 리스크 및 주의사항

### 7.1 SwiftData 마이그레이션

- `UserSettings`에 새 필드 추가 시 반드시 optional 또는 기본값 제공
- `SchedulePreset` 새 @Model 추가 시 lightweight migration 자동 처리
- `PetState` 새 @Model 추가 시 동일
- **테스트 필수**: 기존 데이터 유지 상태에서 앱 업데이트 시 크래시 없어야 함

### 7.2 펫 Progression 설계 원칙

- **punish 금지**: XP 감소, 레벨 강등, 펫 상태 악화 절대 금지
- **성장은 느려도 OK**: 하루 평균 XP 획득량이 ~20-30 XP 수준이면, elder 도달에 약 1달 소요 (자연스러운 장기 동기)
- **펫은 생산성 점수판이 아님**: 펫은 "복귀를 반복한 결과가 남는 시각적 보상"이지, 점수나 랭킹이 아님

### 7.3 메뉴바 공간 제약

- 프리셋 선택과 펫 카드가 모두 드롭다운에 들어가야 함
- 세로 공간 증가 최소화: 펫 카드는 1줄 요약, 프리셋은 드롭다운 확장
- 사용자 빈도가 높은 컨트롤이 위에 위치

### 7.4 성능

- Recovery Review 타임라인: 대량의 AlertingSegment 조회 시 메모리 주의 → 페이지네이션 또는 limit 적용
- PetProgressionService: XP 보상은 SessionTracker 콜백에서 동기 처리 (경량 연산이므로 문제 없을 것)
- SchedulePreset 전환: UserSettings 업데이트 후 즉시 IdleMonitor에 반영되어야 함

### 7.5 테스트 전략

| 기능 | 단위 테스트 | 통합 테스트 |
|------|-----------|------------|
| F1 | RecoveryReviewService 변환 로직, 요약 계산 | 통계 대시보드 ↔ 복귀 리뷰 네비게이션 |
| F2 | 프리셋 CRUD, 마이그레이션, 전환 로직 | QuickControls 프리셋 전환 → IdleMonitor 반영 |
